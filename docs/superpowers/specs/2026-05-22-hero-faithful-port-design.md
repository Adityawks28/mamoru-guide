# Hero Faithful Port — Design Spec

**Date:** 2026-05-22
**Branch:** `feat/aditya/character-system-design-spec`
**Author:** Adityawks28 + Claude

## Summary

The mamoru-guide hero currently shows Port Tower, skyline, mountains, blossoms, and shooters. The Kobe-scene prototype in `Downloads/mamoru_guide_design/` shows substantially more — Crystal Tower, Mosaic Ferris Wheel, Maritime Museum, Akashi-Kaikyo Bridge, City Hall, Harbor Boat, sun rays, drifting clouds, twinkling stars — plus six animations the current site lacks, and uses a different class-naming convention (`.ks-*`) that breaks any direct CSS port.

This spec ports the prototype hero into mamoru-guide as a 1:1 visual match while staying on the existing vanilla TypeScript + Vite stack. React migration is explicitly out of scope and tracked separately.

## Goals

- Visual parity with the prototype `<Hero>` + `<KobeScene>` (screens.jsx:35–364) in both day and night themes.
- Class-name unification with the prototype, so future ports become near-copy-paste.
- A short doc (`docs/hero.md`) that future contributors can use to (a) understand how the hero is assembled and (b) port new sections from the prototype using the same pattern.

## Non-Goals

- React migration. Tracked separately.
- Touching any other section (Earthquake, Vocab, ShowThis, Bag, FirstAid, Shelter, Plan, Typhoon, Drill).
- Re-generating the mascot SVG. Already done in earlier commits via `src/character/mascot.ts`.
- Porting the Tweaks Panel (`tweaks-panel.jsx`). Not in current site, not adding.

## Architecture

A single `<section class="hero arcade-hero">` containing two stacked layers:

**Scene layer** (`aria-hidden="true"`, z-index 1–6)
- `.kobe-scene` — wrapper, full hero bounds, day/night background gradient
  - `.ks-sun-rays` — conic-gradient rays, day-only
  - `.ks-clouds` → `.ks-cloud.c1/c2/c3` — drifting clouds, day-only
  - `.ks-blossoms` → 12× `.ks-petal` — falling petals, day-only
  - `.ks-stars` — generated by `src/stars.ts`, night-only
  - `.ks-shooters` → 4× `.ks-shooter` — gradient trails, night-only
  - `.ks-mountains` — 3-layer SVG ridge (1400×300)
  - `.ks-skyline` — generated by `src/skyline.ts`, contains `.ks-bld` and lit/unlit `.ks-win` windows
  - `.ks-bridge` — Akashi-Kaikyo SVG, far right (480×100)
  - `.ks-cityhall` (`.ch-top`, `.ch-body`) — CSS-only block, left of tower
  - `.ks-foreground` — anchored to hero bounds, ~240px tall
    - `.ks-crystal` — Kobe Crystal Tower SVG (70×240), behind Port Tower
    - `.ks-ferris` — Mosaic Ferris Wheel SVG (80×90), `.ferris-spin` rotates
    - `.ks-museum` — Maritime Museum SVG (120×60)
    - `.ks-tower` — Kobe Port Tower SVG (80×240), `.tower-anim` wrapper
    - `.ks-boat` — Harbor Boat SVG (140×60), drifts + `.boat-wake` pulses
    - `.ks-water` — waterline shelf

**Content layer** (z-index 10)
- `.hero-text` — eyebrow + arcade title + Japanese kanji + sub + CTA buttons
- `.hero-side` — mascot frame (`#mascot` mount) + `.mode-rail` with 3 mode cards

## Documentation Deliverable

A new file `docs/hero.md` containing:

**Part 1 — Prototype → mamoru-guide mapping table.** Row per prototype class/component, showing where each lives in mamoru-guide. Covers all 14 scene elements, both content blocks (`.hero-text`, `.hero-side`), the mascot, mode rail, all CSS vars.

**Part 2 — Asset inventory.** For each of the 14 inline SVG / CSS-only assets: viewBox, source line in screens.jsx, palette tokens used, animation carried, file location in mamoru-guide.

**Part 3 — "How to port a new section" cookbook.** 3 numbered steps:
1. Identify the JSX component in `mamoru_guide_design/screens.jsx` (or other `*-scenes.jsx`).
2. Map its class names: prototype uses `.ks-*` for scene, vocab-specific use their semantic names. Use the mapping table in Part 1 as the canonical reference.
3. For static content, transcribe JSX → HTML. For dynamic content (random arrays via `useMemo`), generate via a corresponding `src/*.ts` file (precedent: `src/skyline.ts`, `src/stars.ts`).

## Risk & Rollback

- **Risk:** rename of `.kobe-tower` → `.ks-tower` might break a CSS selector elsewhere. **Mitigation:** grep all CSS files before commit; the rename was verified hero-only during gap analysis.
- **Risk:** new SVGs increase initial HTML payload by ~8–12KB uncompressed. **Mitigation:** acceptable; matches prototype's payload and is inline (no extra requests).
- **Rollback:** single PR, single commit per logical chunk. Revert if visual diff catches a regression.
